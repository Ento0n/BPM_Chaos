BPM Chaos command runbook
========================

Assumption
----------
Run commands from the project root:

  cd /Users/antonspannagl/PythonProjects/BPM_Chaos

Use the project conda environment:

  conda run -n bpm_chaos python ...


Pipeline order
--------------

1. Create / refresh the synthetic geometry dataset

  conda run -n bpm_chaos python src/create_images.py

Output:

  data/synthetic_bw_256/*.png


2. Train the diffusion model

Convenience script with Mac-friendly defaults:

  bash src/run/train_diffusion_model_20_percent.sh

Direct command:

  conda run -n bpm_chaos python src/train_diffusion_model.py \
    --data-dir data/synthetic_bw_256 \
    --output-dir checkpoints/diffusion \
    --image-size 256 \
    --batch-size 8 \
    --num-workers 4 \
    --max-epochs 10

Useful training knobs:

  --max-epochs 10
  --batch-size 8
  --learning-rate 1e-4
  --accelerator mps
  --accelerator cpu
  --limit-train-batches 1000
  --limit-val-batches 125
  --disable-gradient-checkpointing

Output:

  checkpoints/diffusion/last.ckpt
  checkpoints/diffusion/diffusion-epoch=XX-val_loss=YYYY.ckpt


3. Generate beat images

Use the latest checkpoint:

  conda run -n bpm_chaos python src/generate_diffusion_images.py \
    --checkpoint-dir checkpoints/diffusion \
    --run-parent-dir generated \
    --image-size 256 \
    --num-images 120 \
    --num-inference-steps 100 \
    --seed 42

Use a specific checkpoint:

  conda run -n bpm_chaos python src/generate_diffusion_images.py \
    --checkpoint checkpoints/diffusion/diffusion-epoch=08-val_loss=0.0118.ckpt \
    --run-parent-dir generated \
    --image-size 256 \
    --num-images 120 \
    --num-inference-steps 100 \
    --seed 42

Useful generation knobs:

  --run-parent-dir generated   create generated/YY_MM_DD-HH_MM_SS
  --run-dir generated/name     write into an existing run folder
  --output-subdir diffusion_beats
  --num-images 120           number of beat images
  --image-size 256           must match training size
  --num-inference-steps 100  more steps can improve quality, but is slower
  --seed 42                  repeatable output
  --accelerator mps
  --accelerator cpu

Output:

  generated/YY_MM_DD-HH_MM_SS/diffusion_beats/sample_000.png
  generated/YY_MM_DD-HH_MM_SS/diffusion_beats/sample_001.png
  ...


4. Generate transition frames between beat images

Pixel crossfade baseline:

  RUN_DIR=generated/YY_MM_DD-HH_MM_SS

  conda run -n bpm_chaos python src/generate_crossfade_frames.py \
    --run-dir "$RUN_DIR" \
    --output-subdir frames_pixel \
    --fps 30 \
    --bpm 120 \
    --method pixel \
    --overwrite

Signed-distance-field morph:

  RUN_DIR=generated/YY_MM_DD-HH_MM_SS

  conda run -n bpm_chaos python src/generate_crossfade_frames.py \
    --run-dir "$RUN_DIR" \
    --output-subdir frames_sdf \
    --fps 30 \
    --bpm 120 \
    --method sdf \
    --easing cosine \
    --overwrite

Diffusion noise interpolation:

  This generates beat anchors and in-between frames together from the saved
  diffusion checkpoint. It does not connect already-rendered beat PNGs.
  Each run creates a timestamped child folder named like YY_MM_DD-HH_MM_SS
  under generated, then writes frames, beat images, metadata, and later videos
  into subfolders in that run.

  conda run -n bpm_chaos python src/generate_diffusion_interpolation_frames.py \
    --checkpoint-dir checkpoints/diffusion \
    --run-parent-dir generated \
    --num-beats 120 \
    --image-size 256 \
    --fps 30 \
    --bpm 120 \
    --num-inference-steps 100 \
    --scheduler ddim \
    --interpolation slerp \
    --easing cosine \
    --accelerator mps \
    --require-device mps \
    --seed 42 \
    --overwrite

For 120 BPM at 30 FPS:

  frames_per_beat = round(30 * 60 / 120) = 15

With N beat images and no loop, the output frame count is:

  (N - 1) * frames_per_beat + 1

With N beat images and --loop, the output frame count is:

  N * frames_per_beat + 1

  The final frame is a copy of the first beat image, so the rendered video can
  close on the same picture it starts with.

Useful transition knobs:

  --run-dir generated/YY_MM_DD-HH_MM_SS
  --input-subdir diffusion_beats
  --output-subdir frames
  --fps 30
  --bpm 120
  --frames-per-beat 15       override fps/bpm calculation
  --method pixel             simple image blend
  --method sdf               shape morph via signed distance fields
  --easing linear
  --easing cosine            smoother start/end motion
  --loop                     morph last beat image back to first and close on first
  --mode L                   grayscale output
  --mode RGB
  --overwrite

SDF-specific knobs:

  --threshold 128
  --foreground auto          chooses the smaller black/white region as shape
  --foreground white         white pixels are the shape
  --foreground black         black pixels are the shape
  --sdf-edge-softness 1.5    soft antialiasing edge in pixels
  --sdf-edge-softness 0      hard thresholded SDF edges

Diffusion-interpolation-specific knobs:

  --checkpoint-dir checkpoints/diffusion
  --checkpoint checkpoints/diffusion/name.ckpt
  --run-parent-dir generated
  --frame-subdir frames
  --beat-subdir diffusion_beats
  --num-beats 120            number of diffusion beat anchors
  --scheduler ddim           deterministic and best for interpolation
  --scheduler ddpm           stochastic DDPM-style sampling
  --interpolation slerp      spherical interpolation between noise anchors
  --interpolation lerp       linear interpolation between noise anchors
  --num-inference-steps 100
  --batch-size 4             lower this if MPS/GPU memory is tight
  --ddim-eta 0               keep 0 for deterministic DDIM
  --accelerator mps          use Apple Metal GPU if available
  --require-device mps       fail fast unless torch is really using MPS
  --require-mps              shortcut for --require-device mps
  --no-progress              disable the denoising progress bar
  --no-save-beats            skip generated sample_*.png beat images
  --loop                     interpolate last beat anchor back to first and close on first
  --overwrite                accepted, but timestamped run folders avoid overwrites

Output:

  generated/YY_MM_DD-HH_MM_SS/frames_sdf/frame_000000.png
  generated/YY_MM_DD-HH_MM_SS/frames_sdf/frame_000001.png
  ...

Diffusion interpolation output:

  generated/YY_MM_DD-HH_MM_SS/frames/frame_000000.png
  generated/YY_MM_DD-HH_MM_SS/frames/frame_000001.png
  ...
  generated/YY_MM_DD-HH_MM_SS/metadata.json
  generated/YY_MM_DD-HH_MM_SS/diffusion_beats/sample_000.png
  generated/YY_MM_DD-HH_MM_SS/diffusion_beats/sample_001.png

  metadata.json includes run_started_at, run_finished_at, and total
  generation_elapsed_seconds / generation_elapsed.


5. Create a video or preview GIF

MP4 video, without audio:

  RUN_DIR=generated/YY_MM_DD-HH_MM_SS

  conda run -n bpm_chaos python src/create_video_from_frames.py \
    --run-dir "$RUN_DIR" \
    --frame-subdir frames_sdf \
    --output "$RUN_DIR/videos/crossfade_sdf.mp4" \
    --fps 30 \
    --bpm 120 \
    --overwrite

MP4 video, with audio:

  RUN_DIR=generated/YY_MM_DD-HH_MM_SS

  conda run -n bpm_chaos python src/create_video_from_frames.py \
    --run-dir "$RUN_DIR" \
    --frame-subdir frames_sdf \
    --output "$RUN_DIR/videos/crossfade_sdf_with_audio.mp4" \
    --audio path/to/song.mp3 \
    --fps 30 \
    --bpm 120 \
    --overwrite

Quick GIF preview when ffmpeg is not installed:

  RUN_DIR=generated/YY_MM_DD-HH_MM_SS

  conda run -n bpm_chaos python src/create_video_from_frames.py \
    --run-dir "$RUN_DIR" \
    --frame-subdir frames_sdf \
    --output "$RUN_DIR/videos/crossfade_sdf_preview.gif" \
    --fps 30 \
    --bpm 120 \
    --overwrite

Video from diffusion-interpolated frames:

  Replace YY_MM_DD-HH_MM_SS with the run id printed by the generator.

  conda run -n bpm_chaos python src/create_video_from_frames.py \
    --run-dir generated/YY_MM_DD-HH_MM_SS \
    --output generated/YY_MM_DD-HH_MM_SS/videos/diffusion_interpolation_preview.gif \
    --fps 30 \
    --bpm 120 \
    --overwrite

Useful video knobs:

  --run-dir generated/YY_MM_DD-HH_MM_SS
  --frame-subdir frames
  --beat-subdir diffusion_beats
  --video-subdir videos
  --output generated/YY_MM_DD-HH_MM_SS/videos/name.mp4
  --output generated/YY_MM_DD-HH_MM_SS/videos/name.gif
  --audio path/to/song.mp3   MP4/video output only; requires ffmpeg
  --fps 30
  --bpm 120
  --color-0 111827           color for binary value 0, as RRGGBB or #RRGGBB
  --color-1 F97316           color for binary value 1, as RRGGBB or #RRGGBB
  --frames-per-beat 15
  --loop                     use when the frame generator was run with --loop
  --skip-beat-validation     use frame-dir directly as ordered sequence
  --overwrite

Notes:

  - MP4/MOV/MKV/WebM/AVI output requires ffmpeg in PATH.
  - GIF output uses Pillow and has no audio.
  - create_video_from_frames.py can detect whether the frame directory already
    contains the full sequence including beat images.
  - The current crossfade generator writes a full sequence including beat images.
